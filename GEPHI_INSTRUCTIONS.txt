GEPHI VISUALIZATION GUIDE
=========================
ECE227 — Robot Network Resilience Project
==========================================

WHAT IS GEPHI?
--------------
Gephi is a free, open-source network visualization tool.
It opens the .gexf files generated by this project and produces
publication-quality network diagrams with community colors,
node sizes by degree, and force-directed layouts.

Download: https://gephi.org/users/download/


STEP 1 — GENERATE THE .gexf FILES
-----------------------------------
Run from your project folder:

    python run_all.py

Or just the gephi export:

    python visualizations/gephi_export.py

This creates 4 files in outputs/gephi/:
  • low_homophily_h2.gexf       — h=2  (well-connected, many inter-group edges)
  • high_homophily_h10.gexf     — h=10 (moderate segregation)
  • high_homophily_h20.gexf     — h=20 (high segregation, few cross-group edges)
  • with_calibration.gexf       — h=10 + 3 calibration nodes (gold stars)

Each file encodes:
  - Node color by squad (group attribute)
  - Node size = degree (how many connections)
  - Edge color: blue = inter-group, gray = intra-group
  - Node state: healthy=green, infected=red, calibration=gold


STEP 2 — OPEN IN GEPHI
------------------------
1. Launch Gephi
2. File → Open → navigate to outputs/gephi/ → select a .gexf file
3. Click OK on the import report dialog


STEP 3 — APPLY LAYOUT (ForceAtlas 2)
--------------------------------------
This makes clusters visually separate:

1. In the Layout panel (bottom left), select "ForceAtlas 2"
2. Set these settings:
   - Scaling: 10.0
   - Gravity: 1.0
   - Prevent Overlap: ON
3. Click Run
4. Wait 20–30 seconds until nodes stop moving
5. Click Stop

For high_homophily_h20.gexf you should see 4 tight clusters with
very few lines between them — this is the key visual result.
For low_homophily_h2.gexf clusters will overlap more — showing
the well-connected regime.


STEP 4 — COLOR NODES BY GROUP
-------------------------------
1. In the Appearance panel (top left), click "Nodes" tab
2. Click the color palette icon
3. Select "Partition" → "group"
4. Click Apply
5. The 4 squads will have distinct colors


STEP 5 — SIZE NODES BY DEGREE
-------------------------------
1. Still in Appearance panel, click the resize icon (circles)
2. Select "Ranking" → "Degree"
3. Set min size = 5, max size = 20
4. Click Apply
5. High-degree nodes (hubs) become larger


STEP 6 — EXPORT SCREENSHOTS
-----------------------------
1. File → Export → SVG/PDF/PNG
2. Set width = 2000px, height = 2000px
3. Check "Draw background", set to black (#000000)
4. Uncheck "Draw edges" for a cleaner node-only view (optional)
5. Save as:
   - network_low_h.png
   - network_high_h.png
   - network_with_calibration.png


WHAT TO LOOK FOR IN EACH FILE
------------------------------
low_homophily_h2.gexf:
  → Clusters partially overlap, many blue inter-group edges
  → Shows: in a well-connected swarm, errors diffuse everywhere

high_homophily_h20.gexf:
  → 4 tight, visually distinct clusters with few blue bridges
  → This IS the structural cause of information segregation
  → Caption: "High homophily creates information bottlenecks"

with_calibration.gexf:
  → Same h=10 network but 3 nodes marked gold (calibration)
  → Shows the intervention strategy visually
